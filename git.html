add・commit・branch・merge・push/pull の流れをアニメーションで視覚化。

3 つのエリア — git add & commit の流れ

# ファイルをステージングエリアへ追加 git add feature.js # 特定ファイル git add . # 全変更ファイル git add -p # 変更を chunk 単位で選択 # コミット git commit -m "feat: add feature" # 状態確認 git status git log --oneline --graph
2

ブランチ & マージ — 分岐と合流

# ブランチ操作 git branch # ブランチ一覧 git branch feature # ブランチ作成 git switch feature # ブランチ切り替え (Git 2.23+) git checkout -b feature # 作成 + 切り替え (旧来の方法) # マージ git switch main git merge feature # feature を main にマージ git merge --no-ff feature # マージコミットを必ず作成 # ブランチ削除 git branch -d feature # マージ済みのブランチを削除
3

# リモートを登録 git remote add origin https://github.com/user/repo.git # プッシュ git push origin main # origin の main へ送信 git push -u origin main # 追跡ブランチを設定して push # プル / フェッチ git pull # fetch + merge git fetch # リモートの変更を取得 (mergeしない) git pull --rebase # fetch + rebase（履歴をきれいに保つ）

よく使うコマンド クイックリファレンス

セットアップ
git initリポジトリを初期化
git clone <url>リポジトリをコピー
git config --global user.nameユーザー名を設定
状態確認
git status作業ツリーの状態
git log --onelineコミット履歴（簡易）
git diff変更内容の差分
取り消し
git restore <file>作業ツリーの変更を破棄
git restore --staged <file>ステージングを解除
git revert <hash>コミットを打ち消す
スタッシュ
git stash変更を一時保存
git stash pop最新スタッシュを復元
git stash listスタッシュ一覧
