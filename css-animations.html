@keyframes — Defining Animations

@keyframes bounceBall { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-80px); } } .ball { animation-name: bounceBall; animation-duration: 0.9s; animation-timing-function: ease-in-out; animation-iteration-count: infinite; /* shorthand: name duration timing count */ animation: bounceBall 0.9s ease-in-out infinite; }
2

animation Properties Reference

Property Default Description
animation-namenoneName of the @keyframes to use
animation-duration0sDuration of one cycle (e.g. 1s, 300ms)
animation-timing-functioneaseSpeed curve (ease / linear / cubic-bezier, etc.)
animation-delay0sDelay before start. Negative values start mid-animation
animation-iteration-count1Iteration count. infinite for endless loop
animation-directionnormalPlayback direction — normal / reverse / alternate / alternate-reverse
animation-fill-modenoneState after end — none / forwards / backwards / both
animation-play-staterunningPlay state — running / paused
/* shorthand: name duration timing delay count direction fill play-state */ .element { animation: fadeIn 0.5s ease-out 0.2s 1 normal forwards running; /* common patterns */ animation: spin 2s linear infinite; animation: slideUp 0.4s ease-out forwards; animation: pulse 1.5s ease-in-out 0.5s infinite alternate; }

/* shorthand: property duration timing-function delay */ .element { transition: all 0.3s ease; /* Specify multiple properties individually */ transition: background 0.3s ease, transform 0.2s ease-out; /* Individual properties */ transition-property: background, transform; transition-duration: 0.3s; transition-timing-function: ease; transition-delay: 0s; } /* animation vs transition animation : References @keyframes. Great for auto-play and loops. transition : Responds only to state changes. Works great with :hover and JS class toggles. */
