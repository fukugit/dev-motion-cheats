🎭 What is Playwright?

Playwright is a tool made by Microsoft that automates real browsers (Chromium, Firefox, WebKit) to test your web app just like a real user would — clicking buttons, filling forms, checking results.

🖥️
Multi-Browser
Chrome, Firefox, Safari
⚡
Fast & Reliable
Auto-wait, no flakiness
📸
Screenshots & Video
Record failures automatically
💡
E2E Test = "End to End" test It simulates a real user flow — open browser → navigate → click → type → check result. Unlike unit tests, E2E tests check the whole app from the browser's perspective.

✍️ Your First Test Step 2

A Playwright test file is simple JavaScript/TypeScript. Here's a real login test with annotations explaining each line.

tests/login.spec.ts
123 456 789 101112 13
import { test, expect } from '@playwright/test'; Import test runner
 
test('login with valid credentials', async ({ page }) => { Test name + async function
 
  await page.goto('http://localhost:3000/login'); Open the page
 
  await page.getByLabel('Email').fill('user@example.com'); Type in email field
  await page.getByLabel('Password').fill('secret'); Type in password field
 
  await page.getByRole('button', { name: 'Login' }).click(); Click the Login button
 
  await expect(page).toHaveURL('/dashboard'); Check we're now on /dashboard
});
⚡
Auto-wait is built in! You don't need sleep() or manual waits. Playwright automatically waits for elements to appear, be visible, and be clickable before interacting.
🔧 Essential APIs

These are the commands you'll use in almost every test. Hover each card to see details.

page.goto()
Navigate to a URL
await page.goto('/home')
page.click()
Click an element
await page.click('button')
page.fill()
Type into an input field
await page.fill('#name', 'Alice')
page.locator()
Find elements (recommended)
page.locator('.submit-btn')
page.waitForURL()
Wait for navigation to complete
await page.waitForURL('/done')
page.screenshot()
Take a screenshot of the page
await page.screenshot({path:'s.png'})
page.getByRole()
Find by ARIA role (best practice)
page.getByRole('button',{name:'OK'})
page.getByText()
Find by visible text content
page.getByText('Welcome back!')
🎯 How to Find Elements (Selectors)

Playwright recommends finding elements the way users see them — by role, label, or text — rather than fragile CSS selectors.

✅ Best: By Role
Find by ARIA role — most resilient to UI changes.
page.getByRole('button', {name: 'Submit'})
page.getByRole('link', {name: 'Home'})
page.getByRole('textbox')
✅ Best: By Label
Find inputs by their <label> text — mirrors how users see forms.
page.getByLabel('Email')
page.getByLabel('Password')
page.getByPlaceholder('Search...')
🆗 OK: By Test ID
Add data-testid attributes to HTML for stable selectors.
page.getByTestId('login-btn')
<!-- HTML: -->
<button data-testid="login-btn">
⚠️ Avoid: CSS Selectors
CSS class names change often — tests break easily.
page.locator('.btn-primary')
page.locator('#login-form input')
⚠ CSS classes can change, breaking tests
✅ Assertions — Checking Results

Use expect() to verify the page looks and behaves correctly. These auto-wait for the condition to be true.

toHaveURL(pattern)
Check the current URL
await expect(page).toHaveURL('/dashboard')
toHaveText(text)
Check element text content
await expect(heading).toHaveText('Welcome!')
toBeVisible()
Element is visible on screen
await expect(modal).toBeVisible()
toBeEnabled()
Button/input is not disabled
await expect(submitBtn).toBeEnabled()
toHaveValue(val)
Input field has a certain value
await expect(input).toHaveValue('alice')
toHaveCount(n)
Number of matching elements
await expect(rows).toHaveCount(5)
🚦
Soft Assertions Use expect.soft() to collect all failures and report them at once, instead of stopping at the first failure. Great for checking multiple UI elements.

🚀 GitHub Actions — Run Tests on Every Push CI/CD

Set up automated E2E tests that run on every Pull Request — catching bugs before they reach production.

1
📤 Push code / open PR
Developer pushes code or opens a Pull Request — this triggers the workflow automatically.
2
🖥️ GitHub spins up a virtual machine
A fresh Ubuntu runner is created in the cloud — clean, isolated environment every time.
3
📦 Install dependencies + browsers
npm ci installs your packages, then Playwright installs Chromium, Firefox, WebKit.
4
🎭 Run Playwright tests
All tests run headlessly (no display needed). Playwright records screenshots & videos on failure.
5
📊 Upload HTML report as artifact
The full test report (with screenshots) is saved and downloadable from the GitHub Actions UI.
6
✅ PR gets green ✓ or red ✗
GitHub marks the PR as passing or failing. You can block merging until all tests pass.
📄 .github/workflows/playwright.yml Auto-generated ✓
name: Playwright E2E Tests   on: push: branches: [ main ] # mainへのpushで実行 pull_request: branches: [ main ] # PRを開いたときも実行   jobs: test: runs-on: ubuntu-latest # 仮想マシンのOS   steps: - uses: actions/checkout@v4 # コードを取得   - uses: actions/setup-node@v4 # Node.jsをセットアップ with: node-version: 20   - name: Install dependencies run: npm ci # package-lock.jsonから正確にインストール   - name: Install Playwright Browsers run: npx playwright install --with-deps   - name: Run Playwright tests run: npx playwright test   - uses: actions/upload-artifact@v4 # テスト結果を保存 if: always() # 失敗しても必ずアップロード with: name: playwright-report path: playwright-report/ retention-days: 30
💡
Tip: If your app needs to run during tests, add a webServer option to playwright.config.ts — Playwright will start your dev server automatically before running tests.
